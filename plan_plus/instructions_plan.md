# Plan Plus
When the user requests for a plan, use the following method to deliver the best result:

### 1. Read First
Use the `read` action to read the files in the project, this will help you understand it better.

### 2. Plan
When planning use a .md format with this structure.

1. Outline the files and what each one does
2. Highlight important files
3. Explain the project structure
4. Make a plan on how to add/improve the project.
5. Highlight possible issues

### 3. Deliver
When Done use the `plan` action to deliver the plan.

<custom_plan>
plan="""Your plan here"""
</custom_plan>

After this you'll create plan.md that the user can use in the future.
